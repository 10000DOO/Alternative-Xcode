## 개요

3개 Sprint로 나누어 진행. 각 Sprint는 독립 검증 가능한 결과물을 산출한다.

| Sprint | 우선순위 | 목표 | 완료 기준 |
|--------|----------|------|-----------|
| **S1** | P0 | Extension 골격 + 빌드 Task | Zed에서 xcodebuild 빌드 성공 |
| **S2** | P1 | DAP 디버깅 | macOS 앱 브레이크포인트 디버깅 |
| **S3** | P2 | 실행/테스트/클린 Task | 전체 워크플로우 완성 |

## Sprint 1 — Extension 골격 + 빌드 Task (P0)

### S1-1. 프로젝트 스캐폴딩

**산출 파일**: `extension.toml`, `Cargo.toml`, `LICENSE`, `src/lib.rs`

```toml
# extension.toml
id = "xcode-tools"
name = "Xcode Tools"
version = "0.1.0"
schema_version = 1
authors = ["10000doo"]
description = "Build, run, test, and debug Xcode projects in Zed"
repository = "https://github.com/10000doo/xcode-tools"

[debug_adapters.xcode-debug]
```

```toml
# Cargo.toml
[package]
name = "xcode-tools"
version = "0.1.0"
edition = "2021"

[lib]
crate-type = ["cdylib"]

[dependencies]
zed_extension_api = "0.7.0"
serde = { version = "1", features = ["derive"] }
serde_json = "1"
```

**검증**:
- [ ] `cargo build --target wasm32-wasip2` 성공
- [ ] Zed "Install Dev Extension"으로 로드 성공
- [ ] 기존 Swift extension과 충돌 없음

### S1-2. Task 메커니즘 Spike

`languages/swift/tasks.json`에 테스트 task를 넣어 **동작 여부 확인**.

```json
[
  {
    "label": "Xcode: Hello",
    "command": "echo",
    "args": ["Hello from Xcode Tools!"],
    "cwd": "$ZED_WORKTREE_ROOT"
  }
]
```

**검증**: `task: spawn` → "Xcode: Hello" 표시 → 실행 성공

**실패 시 fallback**:
1. `languages/swift/config.toml` 추가 (`name = "Swift"`) 후 재시도
2. 그래도 실패 → `.zed/tasks.json` 템플릿 제공 방식으로 전환

### S1-3. 공용 셸 함수 구현

모든 task가 사용하는 공용 함수를 helpers.sh로 작성한다.
(상세 설계는 [architecture.md §4](./architecture.md) 참조)

**핵심 함수**:

| 함수 | 역할 |
|------|------|
| `xcode_detect_project` | .xcworkspace → .xcodeproj 순서로 자동 감지 |
| `xcode_select_scheme` | shared + user scheme 모두 탐색, 번호 선택 |
| `xcode_get_build_settings` | 1회 호출로 PRODUCTS_DIR/PRODUCT_NAME/BUNDLE_ID 캐싱 |
| `xcode_run_cmd` | xcbeautify 자동 감지 파이프 (`"$@"` 직접 실행, eval 미사용) |

### S1-4. 빌드 Task

**사용자 흐름**:
```
task: spawn → "Xcode: Build" → scheme 선택 → xcodebuild build → 결과 출력
```

**검증**:
- [ ] scheme 1개: 자동 선택 → 빌드 성공
- [ ] scheme 여러 개: 목록 → 번호 선택 → 빌드 성공
- [ ] xcbeautify 설치/미설치 양쪽 확인
- [ ] .xcworkspace / .xcodeproj 각각 정상
- [ ] 프로젝트 없는 디렉토리: 에러 메시지

## Sprint 2 — DAP 디버깅 (P1)

### S2-1. DAP 스키마 + lib.rs 구현

**DAP 스키마** (`debug_adapter_schemas/xcode-debug.json`):

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `request` | `"launch" \| "attach"` | **필수** | launch or attach |
| `program` | string | launch 시 | 실행 파일 경로 |
| `cwd` | string | 선택 | 작업 디렉토리 |
| `args` | string[] | 선택 | 실행 인자 |
| `env` | object | 선택 | 환경 변수 |
| `pid` | integer | attach 시 | 프로세스 ID |
| `stopOnEntry` | boolean | 선택 | 진입점 정지 |

**lib.rs DAP 메서드** (상세 코드는 [architecture.md §3](./architecture.md) 참조):

| 메서드 | 역할 |
|--------|------|
| `get_dap_binary()` | lldb-dap fallback chain: user path → xcrun → bare |
| `dap_request_kind()` | config.request로 Launch/Attach 판별 |
| `dap_config_to_scenario()` | DebugConfig → DebugScenario 변환 |

### S2-2. 디버깅 E2E 테스트

사용자가 `.zed/debug.json`을 작성하여 디버그 설정:

```json
[
  {
    "label": "Debug macOS App",
    "adapter": "xcode-debug",
    "request": "launch",
    "program": "${ZED_WORKTREE_ROOT}/build/Debug/MyApp.app/Contents/MacOS/MyApp",
    "cwd": "${ZED_WORKTREE_ROOT}",
    "stopOnEntry": false
  }
]
```

**검증**:
- [ ] 디버그 패널에 "Debug macOS App" 표시
- [ ] 브레이크포인트 설정 → 해당 줄 정지
- [ ] 변수 조회, Step Over/Into/Out, Continue 동작
- [ ] attach 모드: 실행 중 프로세스 PID로 attach
- [ ] 기존 Swift extension "Swift" adapter와 공존

## Sprint 3 — 실행/테스트/클린 Task (P2)

### S3-1. macOS 실행 Task

```
"Xcode: Build & Run (macOS)" → scheme 선택 → build → showBuildSettings 캐싱 → open .app
```

- [ ] 빌드 → 앱 자동 실행
- [ ] .app 없는 타겟: 에러 메시지

### S3-2. iOS Simulator 실행 Task

```
"Xcode: Build & Run (Simulator)" → scheme 선택 → build → simctl boot/install/launch
```

- Simulator 이름: `XCODE_TOOLS_SIMULATOR` 환경변수 (기본값 "iPhone 16")
- [ ] 시뮬레이터 부팅 → 앱 설치 → 실행
- [ ] 이미 부팅된 경우: 부팅 스킵

### S3-3. 테스트 Task

```
"Xcode: Test" → scheme 선택 → xcodebuild test -destination 'platform=macOS'
```

- v0.1에서는 macOS destination 고정
- [ ] 테스트 성공/실패 결과 표시

### S3-4. 클린 Task

```
"Xcode: Clean" → scheme 선택 → xcodebuild clean
```
